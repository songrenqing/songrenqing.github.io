    几年前，我自己买过域名，xyz结尾的，一年十几块钱。租用了一个服务器，主机壳的，也是最便宜的那种，大概一百多块钱吧。然后用wordpress搭建了一个静态的网页。

    为了使页面看起来好看，还特意买了深入浅出系列的HTML、CSS，利用下班时间啃完了两本大头书。虽然都是很基础的知识，但是自己本身不是计算机专业，从事的也不是技术开发相关工作，所以学起来比较慢。对于我来说，这算是一个全新的领域。

    后来做完之后还特别得意，毕竟自己从零摸索着一步步实现，特有成就感。其实那就是一个很普通的页面，就像一个刚学会画鸡蛋的小学生画出来的东西。如果非要和专业画家比，那简直是天上和地下的差别。

    当然做这些纯属兴趣，瞎折腾呗。现在技术也越来越成熟，能好好利用现有的技术，对于我来说就够了。

    今天就记录一下我用hexo搭建博客的过程，整个过程很简单。

    hexo快速搭建

    安装

    ➜  ~ npm install -g hexo-cli

    备注：确保之前电脑上成功安装npm

    检查是否安装成功

    ➜  ~ hexo -v

    创建一个文件夹blog，进入并初始化

    ➜  ~ cd /Users/songrenqing/blog
➜  blog hexo init

    备注：文件夹自己随便命名

    安装npm

    ➜  blog npm install

    开启服务

    ➜  blog hexo server

    网页输入http://localhost:4000

    托管到Github

    创建新的仓库

    登录自己的github账号,创建一个新的仓库,名称必须是 user.github.io
    比如我的用户名是songrenqing，新建的仓库就命名为songrenqing.github.io

    修改_config.yml

    进入bolg文件夹中，找到_config.yml，修改deploy内容

    生成静态文件，更新部署

    ➜  blog hexo generate
➜  blog hexo deploy

    输入github的账号和密码

    Username for ‘https://github.com': 填github的注册邮箱
    Password for ‘https://xxxxx@gmail.com':填github的密码

    现在通过 https://songrenqing.github.io 就可以访问网址

    博客的雏型基本上搭建完成，整个过程没有什么难度。如果不喜欢官方的主题，还可以自己更换主题。下一篇文章，讲一下如何更换主题，并加上评论的功能。
